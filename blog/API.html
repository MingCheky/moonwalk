Integrating Moonwalker API in 30 minutes Интеграция Moonwalker API за 30 минут

📅 April 8, 20268 апреля 2026 ⏱ 7 min 👤 Moonwalker ResearchMoonwalker Research

Whether you're building a fintech app, a marketplace, or an internal payout system, Moonwalker API gives you programmatic access to global transfers, multi-currency balances, and real-time FX. In this guide, you'll go from zero to your first successful payout in ~30 minutes. No fluff — just production-ready steps.

What you'll need

  • A Moonwalker Business account (if you don't have one yet, sign up here — KYC takes ~5 minutes).
  • Your test environment: curl, Postman, or your preferred language (Node.js/Python/PHP).
  • API keys — you'll get them from the Dashboard → Developers section.
🔐 Base URL: https://api.moonwalker.com/v1 (sandbox: https://sandbox-api.moonwalker.com/v1)

Step 1 — Retrieve your API keys & set up authentication

Login to your Moonwalker Dashboard, go to Settings → Developers → API keys. Create a new key with payout:write and balance:read scopes. Store your API Key and Secret securely — you'll need them for every request.

Moonwalker uses HMAC-SHA256 signatures. For simplicity, we'll use Basic Auth over TLS (recommended for server‑side only).

# Example: list balances (authenticated)
                            curl -X GET https://api.moonwalker.com/v1/balances \
                            -u "your-api-key:your-api-secret" \
                            -H "Content-Type: application/json"

Response should contain your available balances in USD, EUR, etc. If you see 401, double‑check your credentials and IP whitelist settings.

Step 2 — Create a recipient (payee)

Before sending money, you need a recipient object. POST to /v1/recipients with the payee's details. For bank transfers, include account number, routing/IBAN, and bank country.

curl -X POST https://api.moonwalker.com/v1/recipients \
                            -u "api-key:secret" \
                            -H "Content-Type: application/json" \
                            -d '{
                                "type": "individual",
                                "firstName": "Elena",
                                "lastName": "Rodriguez",
                                "email": "elena@example.com",
                                "bankAccount": {
                                "currency": "EUR",
                                "iban": "DE89370400440532013000",
                                "bankName": "Deutsche Bank"
                                }
                            }'

📌 Save the returned recipientId (e.g. r_abc123) — you'll use it for payouts.

Step 3 — Fund your Moonwalker wallet (optional but required for live)

In sandbox mode, each account receives $10,000 of virtual funds. In production, you can top-up via wire transfer, crypto, or card. Check your balance with GET /v1/balances. Make sure you have enough in the target currency.

Step 4 — Execute a payout (send money)

Now the main event: create a payout. Use the /v1/payouts endpoint. Attach the recipient ID, amount, currency, and a unique idempotencyKey (UUID) to prevent double payments.

curl -X POST https://api.moonwalker.com/v1/payouts \
                        -u "api-key:secret" \
                        -H "Content-Type: application/json" \
                        -d '{
                            "recipientId": "r_abc123",
                            "amount": 1250.00,
                            "currency": "EUR",
                            "description": "Invoice #3345 / product delivery",
                            "idempotencyKey": "e14f5c6a-4b1c-4a2a-9b2e-1a2b3c4d5e6f"
                        }'

The API will return a payoutId and status processing. Most payouts settle within seconds but can take up to 1 business day depending on the destination country and clearing system.

Step 5 — Monitor and handle webhooks

🚀 For a robust integration, set up a webhook endpoint in your Dashboard. Moonwalker will send events like payout.completed, payout.failed, or balance.updated. This is crucial for reconciling transaction statuses without polling.

// Example webhook payload (POST to your URL)
                        {
                        "event": "payout.completed",
                        "data": {
                            "payoutId": "po_987xyz",
                            "status": "completed",
                            "amount": 1250.00,
                            "currency": "EUR",
                            "processedAt": "2026-05-15T14:22:10Z"
                        }
                        }

Testing & next steps

Moonwalker provides a full sandbox environment that mimics production. Use test payment methods — no real money moves. Once your logic is solid, switch to live keys and run a small real transaction to validate the end-to-end flow.

Что понадобится

  • Бизнес‑аккаунт Moonwalker (ещё нет? зарегистрируйтесь, KYC займёт ~5 минут).
  • Тестовое окружение: curl, Postman или язык, который вы используете (Node.js/Python/PHP).
  • API‑ключи — получите их в Дашборде → Разработчики.
🔐 Базовый URL: https://api.moonwalker.com/v1 (песочница: https://sandbox-api.moonwalker.com/v1)

Шаг 1 — получите API‑ключи и настройте аутентификацию

Войдите в панель управления, перейдите в Настройки → Разработчики → API ключи. Создайте ключ с правами payout:write и balance:read. Сохраните API Key и Secret в надёжном месте — они потребуются для каждого запроса.

Moonwalker использует HMAC-SHA256, но для простоты в этом руководстве применяется Basic Auth через TLS (только для серверной стороны).

# Пример: получить балансы (аутентификация)
                            curl -X GET https://api.moonwalker.com/v1/balances \
                            -u "ваш-api-ключ:ваш-секрет" \
                            -H "Content-Type: application/json"

В ответе вы увидите доступные остатки в USD, EUR и т.д. Если получаете 401, проверьте ключи и белый список IP.

Шаг 2 — Создайте получателя

Перед отправкой нужен объект получателя. Отправьте POST /v1/recipients с данными. Для банковского перевода укажите номер счёта, IBAN и страну банка.

curl -X POST https://api.moonwalker.com/v1/recipients \
                            -u "api-key:secret" \
                            -H "Content-Type: application/json" \
                            -d '{
                                "type": "individual",
                                "firstName": "Елена",
                                "lastName": "Родригес",
                                "email": "elena@example.com",
                                "bankAccount": {
                                "currency": "EUR",
                                "iban": "DE89370400440532013000",
                                "bankName": "Deutsche Bank"
                                }
                            }'

📌 Сохраните полученный recipientId (например r_abc123) — он понадобится для выплат.

Шаг 3 — Пополните кошелёк (опционально для боевого режима)

В песочнице каждый аккаунт получает $10 000 виртуальных средств. В боевом режиме пополнение происходит через банковский перевод, криптовалюту или карту. Проверьте баланс через GET /v1/balances. Убедитесь, что в валюте выплаты достаточно средств.

Шаг 4 — Выполните выплату

Главное событие: создание выплаты. Используйте эндпоинт /v1/payouts. Передайте recipientId, сумму, валюту и уникальный idempotencyKey (UUID) для защиты от дублей.

curl -X POST https://api.moonwalker.com/v1/payouts \
                            -u "api-key:secret" \
                            -H "Content-Type: application/json" \
                            -d '{
                                "recipientId": "r_abc123",
                                "amount": 1250.00,
                                "currency": "EUR",
                                "description": "Инвойс #3345 / поставка товара",
                                "idempotencyKey": "e14f5c6a-4b1c-4a2a-9b2e-1a2b3c4d5e6f"
                            }'

API вернёт payoutId и статус processing. Большинство выплат зачисляются в течение нескольких секунд, но в некоторых странах может потребоваться до 1 рабочего дня.

Шаг 5 — Мониторинг и вебхуки

🚀 Настройте вебхук в Дашборде. Moonwalker будет отправлять события: payout.completed, payout.failed, balance.updated. Это избавит от постоянного опроса статуса и упростит сверку.

                            {
                            "event": "payout.completed",
                            "data": {
                                "payoutId": "po_987xyz",
                                "status": "completed",
                                "amount": 1250.00,
                                "currency": "EUR",
                                "processedAt": "2026-05-15T14:22:10Z"
                            }
                            }

Что дальше?

Песочница Moonwalker полностью имитирует боевую среду. Используйте тестовые методы оплаты — реальные деньги не списываются. Когда логика готова, переключитесь на боевые ключи и проведите небольшой платёж для проверки.
